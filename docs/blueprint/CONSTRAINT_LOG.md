## NFR Constraints (from Contract Section 7, architectural impact only)

**CONSTRAINT-010 | LINE webhook must respond within 3000ms (NFR-001)**
- Source: NFR-001, BR-002
- Decision: `io_ack_line_webhook` sends HTTP 200 synchronously before any processing. Processing is enqueued asynchronously.
- Rationale: LINE platform drops webhook if no 200 within 3s.
- Scope: `src/channel/channel_manager.ts`
- Forbidden: Any blocking operation before the HTTP 200 response

**CONSTRAINT-011 | MCP server bound to 127.0.0.1 loopback only (NFR-005)**
- Source: NFR-005
- Decision: MCP Express app calls `app.listen(port, '127.0.0.1')`. Not `0.0.0.0`.
- Rationale: MCP must not be externally accessible per security requirement.
- Scope: `src/mcp/mcp_server.ts`
- Forbidden: Binding MCP to 0.0.0.0 or external interface

**CONSTRAINT-012 | All WebSocket connections require JWT auth before any data processing (NFR-004)**
- Source: NFR-004, FR-032
- Decision: Auth check runs in the `upgrade` event handler before the WebSocket connection is accepted. Unauthenticated connections are rejected at the HTTP upgrade stage.
- Rationale: 100% enforcement means no data reaches the gateway without auth.
- Scope: `src/channel/channel_manager.ts`, `src/auth/auth_manager.ts`
- Forbidden: Accepting WebSocket connection then checking auth async

**CONSTRAINT-013 | LINE webhook signature checked before any message processing (NFR-006)**
- Source: NFR-006, FR-003, EC-001, BR-001
- Decision: `validate_line_signature` runs as Express middleware before the route handler. Invalid signatures return 401 immediately.
- Rationale: BR-001 requires rejection before processing; middleware enforces this.
- Scope: `src/channel/channel_manager.ts`
- Forbidden: Signature check after route handler logic begins

**CONSTRAINT-014 | Heartbeat interval configurable via env var (NFR-007)**
- Source: NFR-007, FR-016
- Decision: `HEARTBEAT_INTERVAL_MS` env var controls interval. Default: 300000ms. Grace period: `STARTUP_GRACE_MS` = 60000ms. Channel grace: `CHANNEL_CONNECT_GRACE_MS` = 120000ms.
- Rationale: Contract specifies configurable interval.
- Scope: `src/heartbeat/heartbeat_monitor.ts`
- Forbidden: Hardcoded 300000 in function body; use named constants

**CONSTRAINT-015 | Minimum 80% test coverage (Contract S8)**
- Source: Contract S8 automation defaults
- Decision: Jest coverage threshold set to 80% globally. `--coverage` flag required on all test runs.
- Rationale: Contract confirmed 80% minimum.
- Scope: `jest.config.ts`, CI config
- Forbidden: Skipping coverage; threshold below 80%

## Open Question Resolutions (from SDD Section 8)

- Decision: `select_messages_for_compaction` uses `keepRecent = parseInt(process.env.COMPACTION_KEEP_RECENT ?? '20')`.

**CONSTRAINT-017 | Compaction queue timeout is 30 seconds (OQ-002)**
- Source: SDD S8/OQ-002
- Decision: `queue_llm_call_during_compaction` rejects with `CompactionTimeoutError` after 30000ms.
- Rationale: Best-practice default to prevent indefinite queuing.

**CONSTRAINT-018 | Only LINE `message` event type supported in MVP (OQ-003)**
- Source: SDD S8/OQ-003
- Decision: `transform_line_event_to_message` processes only events where `event.type === 'message'`. Other event types are logged and ignored.
- Rationale: Best-practice: scoped MVP; other types out of scope.
- Scope: `src/channel/channel_manager.ts`
- Forbidden: Processing follow/unfollow/postback events in MVP

**CONSTRAINT-019 | SkillStore persisted as JSON file at `~/.aaos/skills/registry.json` (OQ-004)**
- Source: SDD S8/OQ-004
- Decision: SkillStore reads/writes `${OPENCLAW_WORKSPACE}/skills/registry.json`. Loaded at startup.
- Rationale: File-based persistence consistent with overall storage constraint.
- Scope: `src/skills/skill_manager.ts`
- Forbidden: In-memory-only SkillStore; SQLite

**CONSTRAINT-020 | SessionStore is in-memory for MVP; warm-start from JSONL (OQ-005)**
- Source: SDD S8/OQ-005
- Decision: Sessions stored in a `Map<string, Session>` in memory. On startup, existing JSONL files in `~/.aaos/sessions/` are scanned to reconstruct session metadata.
- Rationale: Simpler for MVP; JSONL files provide warm-start recovery.
- Scope: `src/channel/channel_manager.ts`
- Forbidden: Writing session index to disk in MVP

**CONSTRAINT-021 | ACP pipeline max 10 stages (OQ-006)**
- Source: SDD S8/OQ-006
- Decision: `chain_agent_run_stages` validates `stages.length <= 10` before execution. Throws `PipelineLimitError` if exceeded.
- Rationale: Best-practice guard against runaway pipelines.
- Scope: `src/acp/acp_runtime.ts`
- Forbidden: Unbounded stage arrays

**CONSTRAINT-022 | Heartbeat subsystem ping timeout is 10 seconds (OQ-007)**
- Source: SDD S8/OQ-007
- Decision: Each subsystem ping in `ping_all_subsystems` has a 10000ms timeout via `Promise.race`. Timeout = degraded.
- Rationale: Best-practice; fast enough to detect failures within one heartbeat cycle.
- Scope: `src/heartbeat/heartbeat_monitor.ts`
- Forbidden: No timeout on subsystem pings

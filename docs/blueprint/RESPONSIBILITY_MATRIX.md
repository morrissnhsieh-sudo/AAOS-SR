## Violation Rules (for /codeverify)

**VIOLATION-001 | Auth logic found outside auth_manager.ts**
- Detection: JWT verify/sign or HMAC computation in any file other than `src/auth/auth_manager.ts`
- Auto-fix: Extract to auth_manager, replace with import + function call
- Severity: HIGH

**VIOLATION-002 | File I/O (fs calls) outside designated owner files**
- Detection: `fs.readFile`, `fs.writeFile`, `fs.appendFile` outside `memory_system.ts` or `skill_manager.ts`
- Auto-fix: Extract to `io_<n>()` in the owner file; update caller
- Severity: HIGH

**VIOLATION-003 | Session state mutation outside channel_manager.ts**
- Detection: Direct SessionStore write in any file other than `channel_manager.ts`
- Auto-fix: Replace with call to `update_session_last_active` or `get_or_create_session`
- Severity: HIGH

**VIOLATION-004 | Tool registry mutation outside tool_dispatcher.ts**
- Detection: Direct ToolRegistry write in any file other than `tool_dispatcher.ts`
- Auto-fix: Replace with call to `register_tool` or `deregister_skill_tools`
- Severity: HIGH

**VIOLATION-005 | npm child_process spawn outside skill_manager.ts**
- Detection: `child_process.spawn` or `exec` with `npm` in any file other than `skill_manager.ts`
- Auto-fix: Extract to `io_run_npm_install` in skill_manager; replace with call
- Severity: HIGH

**VIOLATION-006 | LLM provider call outside plugin_engine.ts**
- Detection: Direct Anthropic/Ollama/Google SDK call outside `plugin_engine.ts`
- Auto-fix: Route through Plugin Engine API
- Severity: HIGH

**VIOLATION-007 | Heartbeat timer creation outside heartbeat_monitor.ts**
- Detection: `setInterval` with heartbeat-like interval outside `heartbeat_monitor.ts`
- Auto-fix: Consolidate into `schedule_heartbeat`
- Severity: MEDIUM

**VIOLATION-008 | Business logic in I/O functions**
- Detection: Non-trivial branching or transformation logic inside `io_<n>()` functions
- Auto-fix: Extract business logic to a `transform_<n>()` or validate caller
- Severity: MEDIUM

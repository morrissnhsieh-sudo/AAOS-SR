## Section 1 ??Executive Summary

AAOS (Autonomous Agent Orchestration System) is a persistent, self-hosted personal AI assistant platform. Running as a Node.js (TypeScript) process inside a Docker container, it acts as a unified gateway connecting messaging channels (LINE, browser Control UI), AI language model providers, skills, plugins, and file-based memory. All user requests are received, processed, and responded to by the gateway, enabling stateful, tool-augmented conversations.

The system targets a single-user deployment model. Success is measured by message round-trip response time: a user message receives a correct, tool-augmented response within 5 seconds under normal load.

---

## Section 2 ??Problem Statement

Personal AI assistants typically require cloud infrastructure, lack extensibility, and offer no persistent memory across conversations. AAOS solves this by providing a self-hosted gateway that:
- Unifies multiple messaging channels into a single processing pipeline
- Extends LLM capabilities via installable npm-based skills
- Persists knowledge through file-based memory without model training
- Supports multiple LLM providers with automatic failover

## Section 4 ??User Stories

### US-001: Message Processing Pipeline (maps to FR-001 through FR-011, WF-01)
**As an End User**, I want to send a message via LINE or the Control UI so that I receive an AI-generated response, optionally augmented by tool results, back in the same channel.

**Acceptance Criteria:**

**AC-001** (LINE webhook receipt ??FR-001, EC-001)
```gherkin
Given a LINE platform webhook POST to /webhook/line
When the X-Line-Signature header is present and valid
Then the gateway Returns HTTP 200 immediately and processes the message asynchronously
```

**AC-002** (LINE signature validation ??FR-003, EC-001, BR-001)
```gherkin
Given a LINE platform webhook POST to /webhook/line
When the X-Line-Signature header is missing or invalid
Then the gateway Rejects the request with HTTP 401 { reason: "Missing signature" } and logs the attempt
```

**AC-003** (Message normalization ??FR-004, EC-002)
```gherkin
Given a channel adapter has received a raw message
When the message is missing required fields after normalization
Then the gateway Rejects the message, logs the error, and does not route it to the Agent Runner
```

**AC-004** (Session routing ??FR-005)
```gherkin
Given a normalized message with a valid channel_id and user_id
When the Channel Manager processes the message
Then the gateway Returns the existing session or creates a new one and routes the message to it
```

**AC-005** (System context injection ??FR-006, EC-013)
```gherkin
Given an Agent Runner starts a new run for a session
When HEARTBEAT.md, BOOT.md, and MEMORY.md exist in the workspace
Then the gateway Prepends all three files to the LLM prompt
```

**AC-006** (Missing context file ??FR-006, EC-013)
```gherkin
Given an Agent Runner starts a new run
When BOOT.md or HEARTBEAT.md is missing from the workspace
Then the gateway Logs a warning, proceeds with available files, and does not abort the run
```

**AC-007** (Text response delivery ??FR-007, FR-011)
```gherkin
Given an LLM returns a plain text response
When no tool calls are present in the response
Then the gateway Delivers the text response to the user via the originating channel
```

**AC-008** (Tool call dispatch ??FR-008, FR-009, EC-004)

**AC-009** (Unknown tool call ??FR-009, EC-004)

**AC-010** (Parallel tool calls ??FR-008, FR-009, EC-015)

**AC-011** (WS receipt and auth ??FR-002, FR-032, EC-010)
```gherkin
Given a browser Control UI client initiates a WebSocket connection to /ws/chat

```

**AC-012** (Expired WS token ??FR-032, EC-010)

**AC-013** (WS request-response correlation ??FR-033)

**AC-014** (WS server-push events ??FR-034, EC-009)

### US-003: Heartbeat & Health Monitoring (maps to FR-016 through FR-020, FR-037, WF-03)


**AC-025** (Subsystem degraded ??FR-016, EC-012)
```gherkin

```

**AC-026** (Channel keepalive ??FR-017)
```gherkin

Then the gateway Sends keepalive frames to all active connections
```

**AC-027** (Channel grace period ??FR-020, BR-006)
```gherkin

When the channel fails to connect within 120 seconds
Then the gateway Marks the channel as "degraded" (not terminated)
```

**AC-028** (Startup grace period ??FR-037)
```gherkin

Then the gateway Does not activate health monitoring
```

**AC-029** (Log flush ??FR-019, BR-008)
